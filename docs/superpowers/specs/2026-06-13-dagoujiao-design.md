# 大狗叫 · 设计文档

**日期：** 2026-06-13
**状态：** 待审批

---

## 一、概述

一个魔性洗脑的网页。用户点击动物让它叫，声音叠加，计数累加。大狗叫满 100 次解锁隐藏角色哈基米。连续狂点会触发死亡机制——狗/鸡被"叫死"，Game Over 清零。

**目标：** 抖音传播，截图可发朋友圈。

---

## 二、三个模块（标签切换）

| 模块 | 初始状态 | 解锁条件 |
|------|---------|---------|
| 🐕 大狗叫 | ✅ 可见 | 无需解锁 |
| 🐔 叮咚鸡 | ✅ 可见 | 无需解锁 |
| 🐱 哈基米 | 🔒 锁定 | 大狗叫满 100 次解锁 |

解锁后顶部导航从 2 个标签变成 3 个。

---

## 三、大狗叫模块

### 交互
- 点狗图片 → 狗叫一声 → 计数 +1 → 显示「大狗叫了 N 次」
- 连续狂点 → 多个声音同时叠加播放
- 满 100 次 → 永久解锁 🔓 哈基米标签，继续计数

### 死亡机制
- **检测：** 连续快速点击 20 次（每次间隔 < 1.5 秒算连续）
- **第 20 次：** 弹出提示「大狗叫有点不舒服😰」
- **用户继续点：** 触发 Game Over
  - 狗图片切换为"死狗"图片
  - 大狗叫计数清零
  - 全屏显示「大狗叫被你叫死了💀」
  - 声音停止，不能再点
- **用户在第 20 次后停止：** 5 秒不点自动恢复，狗恢复正常（图片变回来，下次连续点又从 0 开始检测）
- 恢复后可以继续玩，下次连续 20 次会再次触发

### 用户自定义
- 正常狗图片：`static/images/dog.png`
- 死狗图片：`static/images/dog-dead.png`
- 狗叫声：`static/sounds/dog-bark.mp3`
- 或通过页面上传

## 四、叮咚鸡模块

与大狗叫**完全相同的机制**：

- 正常鸡图片：`static/images/chicken.png`
- 死鸡图片：`static/images/chicken-dead.png`
- 鸡叫声：`static/sounds/chicken-cluck.mp3`
- 20 次连续 → 警告 → Game Over → 「叮咚鸡被你叫死了💀」
- 恢复机制同大狗叫

## 五、哈基米模块

- **无任何限制** —— 没有死亡机制，没有上限
- 随便点，随便叫，不会死
- 猫咪图片：`static/images/cat.png`
- 猫叫声：`static/sounds/cat-meow.mp3`

## 六、计数系统

- 三个模块各有独立计数器
- 数据存储在浏览器 localStorage，刷新不丢失
- 显示格式：「[名称]叫了 N 次」

## 七、技术架构

```
用户浏览器
    │
    ▼
Flask → 渲染 index.html
    │
    ├── /static/images/   ← 用户替换图片
    ├── /static/sounds/   ← 用户替换音频
    └── 前端 JS 处理所有交互
```

**前端：** 单 HTML + CSS + JS。Web Audio API 播放声音。localStorage 存计数。
**后端：** Flask 只负责渲染页面，不涉及业务逻辑。
**部署：** Railway（gunicorn）

## 九、Game Over 界面

```
┌──────────────────────────────────┐
│                                  │
│          💀                      │
│     大狗叫被你叫死了              │
│                                  │
│     [重新开始] 按钮               │
│                                  │
└──────────────────────────────────┘
```

点重新开始 → 计数清零 → 恢复狗图片 → 可以重新玩。哈基米解锁状态保留（不因 Game Over 而重新锁定）。

## 十、不做

- 不保存排行榜
- 不登录不注册
- 不做分享功能（用户自己截图）
- 哈基米不被叫死
